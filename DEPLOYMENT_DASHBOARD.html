‚ö° Heroku Deployment

‚ÑπÔ∏è Fastest option: 5-10 minutes to live backend

Step 1: Authenticate

Open PowerShell and run:

heroku login -i

Enter your Heroku email and password

Step 2: Create App

cd c:\Users\tmoore\What-Is-Life-LifeQuest
heroku apps:create lifequest-api

Note: If the app name is taken, Heroku will suggest an alternative

Step 3: Deploy Code

git push heroku main

This pushes your code and automatically builds the Docker image

Step 4: (Optional) Add MongoDB

heroku config:set MONGODB_URI="mongodb+srv://user:pass@cluster0.mongodb.net/lifequest" --app=lifequest-api

Paste your MongoDB Atlas connection string here

Step 5: View Deployment

heroku logs --tail --app=lifequest-api

Watch the logs in real-time. Once complete, you'll see:

üöÄ LifeQuest Backend Server running on port 3001
‚úÖ Your URL: https://lifequest-api.herokuapp.com

Step 6: Update Frontend

Add to .env.local:

REACT_APP_API_URL=https://lifequest-api.herokuapp.com

üöÄ Automated Script (Windows)

Or run the automated PowerShell script:

powershell -NoProfile -ExecutionPolicy Bypass -File DEPLOY_HEROKU.ps1
‚ö†Ô∏è Important: If you haven't logged in to Heroku before, run heroku login first

‚òÅÔ∏è AWS Elastic Beanstalk Deployment

‚ÑπÔ∏è Production-grade deployment with auto-scaling

Prerequisites

Install required tools:

pip install awsebcli --user
npm install -g aws-cli

Step 1: Configure AWS Credentials

aws configure

Enter your AWS Access Key ID, Secret Access Key, default region (us-east-1), and output format (json)

Step 2: Initialize Elastic Beanstalk

cd c:\Users\tmoore\What-Is-Life-LifeQuest
eb init -p "Node.js 18 running on 64bit Amazon Linux 2" lifequest-api --region us-east-1

Step 3: Create Environment

eb create lifequest-env --instance-type t3.micro

This creates an EC2 instance. Takes 3-5 minutes.

Step 4: Set Environment Variables

eb setenv MONGODB_URI="mongodb+srv://user:pass@cluster.mongodb.net/lifequest"

Step 5: Deploy Application

eb deploy

Step 6: Get Deployment URL

eb open --print-url
‚úÖ Your backend is now live with auto-scaling!

Useful Commands

eb status
eb logs --all
eb terminate

üóÑÔ∏è MongoDB Atlas Setup

‚ÑπÔ∏è Free tier: 512MB storage, perfect for indie games

Step 1: Create Account

Visit https://cloud.mongodb.com and:

  • Click "Sign Up"
  • Create account (email or GitHub)
  • Verify email

Step 2: Create Free Cluster

  • Click "Create" ‚Üí "Build a Database"
  • Choose Shared Tier (Free)
  • Select AWS as provider
  • Choose closest region
  • Click "Create Cluster"

Takes 3-5 minutes to provision

Step 3: Create Database User

  • Click "Database Access" in left sidebar
  • Click "Add New Database User"
  • Username: lifequest
  • Generate secure password (or create one)
  • Select "Built-in Role" ‚Üí "Atlas admin"
  • Click "Add User"

Step 4: Configure Network Access

  • Click "Network Access" in left sidebar
  • Click "Add IP Address"
  • Choose 0.0.0.0/0 (allow anywhere)
  • Click "Confirm"

‚ö†Ô∏è For production: whitelist only your app's IP

Step 5: Get Connection String

  • Click your cluster name
  • Click "Connect"
  • Select "Connect your application"
  • Copy the connection string

Step 6: Use Connection String

Replace placeholders in the string:

<username> ‚Üí lifequest
<password> ‚Üí your_generated_password
myFirstDatabase ‚Üí lifequest

Example:

mongodb+srv://lifequest:YourPassword123@cluster0.abcd.mongodb.net/lifequest?retryWrites=true&w=majority

Step 7: Set in Your Host

For Heroku:

heroku config:set MONGODB_URI="your_connection_string" --app=lifequest-api

For Railway:

Set MONGODB_URI in Dashboard ‚Üí Variables

For AWS EB:

eb setenv MONGODB_URI="your_connection_string"

For Local:

echo MONGODB_URI=your_connection_string >> .env
node backend/server-enhanced.js
‚úÖ Verify with: curl https://your-backend/health

Should show: "database": "MongoDB"

üê≥ Docker Local Development

‚ÑπÔ∏è Run backend + MongoDB locally with Docker

Prerequisites

Install Docker Desktop:

  • Windows/Mac: https://www.docker.com/products/docker-desktop
  • Linux: Install Docker Engine via package manager
  • Verify: docker --version

Step 1: Start Docker Compose

cd c:\Users\tmoore\What-Is-Life-LifeQuest
docker compose up --build -d

The -d flag runs in the background

Step 2: Verify Services

docker compose ps

Should show both services running:

  • lifequest-mongo (MongoDB on port 27017)
  • lifequest-backend (Backend on port 3001)

Step 3: Test Health Endpoint

curl http://localhost:3001/health

Should return JSON with "database": "MongoDB"

Step 4: View Logs

docker compose logs -f backend

Watch backend logs in real-time

Step 5: Test API Endpoints

curl http://localhost:3001/api/leaderboard
curl http://localhost:3001/api/achievements
curl http://localhost:3001/api/lore

Stop Services

docker compose down

This stops and removes containers (data persists)

‚úÖ Now test the frontend at http://localhost:3000

Click the Dev/Test buttons to verify save/load functionality
